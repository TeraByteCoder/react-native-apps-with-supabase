Generierte User Storys

Diese Storys wurden aus Aufgabe, Prototyp und UI-Wunsch abgeleitet und vom Manager an Worker-Subagents bzw. fokussierte Umsetzungsschritte übergeben.

US-01Mustui-worker

Finished dark user flow

Als Visitor/Athlete möchte ich Landing, Login, Register, Dashboard und Health in einer fertigen App nutzen.

  • Reiner schwarzer Hintergrund, weiße Schrift, hellblaue Primary Buttons.
  • Nutzer kann zwischen allen Screens navigieren.
  • Login/Register zeigen lokale Validierung und Feedback.
US-02Mustui-worker

Shared dashboard theme alignment

Als Athlete möchte ich ein einheitliches Dashboard ohne alten gelb/oliven Look.

  • Shared Theme nutzt Schwarz, Weiß, Grau und Hellblau.
  • WorkoutCard und Dashboard wirken zusammenhängend.
  • Storybook-Varianten bleiben lauffähig.
US-03Shouldui-worker

Storybook remains reviewable

Als Developer möchte ich die UI-Zustände weiterhin isoliert in Storybook prüfen.

  • WorkoutTrackingDashboard enthält Synced, Syncing, Offline, Error.
  • WorkoutCard enthält Default, Beginner, Advanced.
  • Der Storybook Build läuft erfolgreich.
US-04Mustbackend-ready

Supabase-safe demo setup

Als Nutzer möchte ich eine App, die jetzt läuft und später echte Supabase-Daten lesen kann.

  • Public URL ist dokumentiert, Anon-Key bleibt Placeholder.
  • Mock-Fallback ist sichtbar und verhindert leere Screens.
  • Keine persönlichen Access Tokens werden im Repo gespeichert.

Verwendung

So wird das Storyboard benutzt

Die Website ist statisch und braucht keinen Build. App und Storybook bleiben separat ausführbar.

Storyboard öffnen

Direkt im Browser öffnen.

xdg-open docs/storyboard/index.html

Workout-App starten

Interaktive React-Native-Web App ausführen.

npm run dev --workspace @workout/workout-app -- --web --port 8081

Storybook starten

Komponenten und Varianten isoliert prüfen.

npm run storybook
